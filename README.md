# FitDiary — Веб-приложение для учета тренировок и питания

Приложение предназначено для отслеживания личного фитнес-прогресса, ведения дневника питания и записи тренировочных сессий.

## Описание проекта

FitDiary — это комплексный инструмент, который позволяет пользователям:
1.  **Отслеживать питание:** Вести журнал приемов пищи с автоматическим расчетом калорийности на основе базы продуктов.
2.  **Управлять тренировками:** Создавать тренировочные сессии, добавлять упражнения и фиксировать подходы (вес и повторения).
3.  **Контролировать прогресс:** Записывать замеры тела (вес, талия, бедра) и отслеживать их динамику.

## Архитектура базы данных

Приложение использует сложную реляционную базу данных, состоящую из **11 таблиц**, связанных отношениями "один-ко-многим":

**Пользователь и Прогресс:**
1.  `users` — Данные авторизации и профиль.
2.  `measurements` — Журнал замеров тела.

**Модуль Питания:**
3.  `meal_types` — Справочник типов приема пищи (Завтрак, Обед, Ужин).
4.  `food_categories` — Справочник категорий продуктов (Молочные, Мясо, Фрукты).
5.  `products` — База продуктов с макронутриентами (белки, жиры, углеводы, калории).
6.  `food_entries` — Журнал съеденного, связанный с пользователем, продуктом и типом приема пищи.

**Модуль Тренировок:**
7.  `muscle_groups` — Справочник групп мышц (Грудь, Спина, Ноги).
8.  `exercises` — Справочник упражнений, привязанных к группам мышц.
9.  `workouts` — Тренировочные сессии пользователя.
10. `workout_items` — Конкретные упражнения, выполняемые в рамках одной тренировки.
11. `workout_sets` — Подходы, повторения и рабочие веса для каждого упражнения.

## Ключевые особенности и UX-решения

### 1. Работа с человекочитаемыми данными
В соответствии с требованием *"информация из внешних словарей должна отображаться как читаемый текст, а не коды внешней таблицы"*:
*   **Ввод:** В формах используются выпадающие списки (`collection_select`) или группированные селекты. Пользователь выбирает "Куриная грудка" из списка, система сохраняет `ID` в фоновом режиме.
*   **Вывод:** Во всех таблицах отображаются связанные названия (например, отображается название категории "Белки" вместо `category_id: 2`).

### 2. Вложенные формы и логика
*   **Создание продуктов:** Пользователь может добавить новый продукт с КБЖУ вручную прямо в форме записи приема пищи (используется механизм Nested Attributes).
*   **Сложные тренировки:** Страница тренировки позволяет добавлять множество упражнений и фиксировать подходы в рамках одного экрана без перезагрузки страницы.

## Установка и запуск

Для локального запуска убедитесь, что установлены Ruby и Rails.

1.  **Клонирование репозитория:**
    ```bash
    git clone https://github.com/your-username/fit_diary.git
    cd fit_diary
    ```

2.  **Установка зависимостей:**
    ```bash
    bundle install
    ```

3.  **Настройка базы данных:**
    Эта команда создаст БД, выполнит миграции и заполнит начальные данные (справочники).
    ```bash
    rails db:setup
    ```

4.  **Запуск сервера:**
    ```bash
    rails server
    ```

5.  **Вход в приложение:**
    Откройте браузер по адресу `http://localhost:3000`.
